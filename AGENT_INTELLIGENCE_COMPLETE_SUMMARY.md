## 🎯 项目概述

本项目将 Agent 系统从基础的多Agent架构升级为具备深度语义理解、主动智能、多模态交互和持续学习能力的智能化系统。

## 🚀 核心功能详解

### 1. 语义理解与RAG

#### 向量存储服务
- **1536维向量存储**: 支持语义级数据存储
- **余弦相似度搜索**: 精准语义匹配
- **本地Embedding生成**: API + 本地降级双方案
- **自动容量管理**: 最大500向量

#### RAG服务
- **8个预设设计案例**: 覆盖主流设计场景
- **语义检索**: 基于用户输入自动检索相关案例
- **Prompt增强**: 自动注入相关案例
- **智能风格推荐**: 基于语义相似度

#### 意图识别服务
- **11种意图类型**: 创建、修改、询问、确认等
- **双引擎识别**: 规则匹配 + AI语义理解
- **实体提取**: 自动提取设计类型、风格、受众
- **置信度评估**: 低置信度时主动澄清

#### Prompt构建器
- **Token预算管理**: 自动估算和控制
- **Few-shot示例**: 12个精选示例
- **多维度增强**: RAG + 记忆 + 意图

### 2. 主动智能

#### 行为预测服务
- **8种行为类型**: 消息、风格选择、图像生成等
- **用户画像**: 偏好Agent、风格、任务类型
- **趋势分析**: 风格趋势、任务趋势
- **智能预测**: 下一步行为预测

#### 主动建议引擎
- **6种建议类型**: 风格、Agent、任务、操作、内容、快捷
- **多维度生成**: 行为预测、RAG、记忆、上下文
- **智能过滤**: 去重、置信度过滤、优先级排序

#### 智能工作流引擎
- **7种节点类型**: 开始、分析、设计、评审、修订、确认、结束
- **2个预置模板**: IP形象设计、品牌设计
- **自动化执行**: 支持自动和手动节点
- **质量检查**: 自动评分和改进建议

### 3. 多模态交互

#### 语音服务
- **语音识别(STT)**: Web Speech API实时识别
- **语音合成(TTS)**: 浏览器原生合成
- **9种语音命令**: 创建、切换、确认、取消等
- **语音对话模式**: 听+说完整流程

#### 图像理解服务
- **图像内容分析**: 物体、颜色、风格、氛围识别
- **风格分析**: 设计风格识别和推荐
- **草图理解**: 手绘草图理解
- **颜色提取**: 主色调和配色方案

### 4. 持续学习

#### 反馈闭环服务
- **7种反馈类型**: 显式点赞/点踩、隐式停留、纠正、跳过等
- **满意度分析**: 趋势、问题、优势识别
- **学习数据生成**: 成功/失败模式识别
- **优化建议**: 自动生成改进建议

## 💡 使用示例

### 初始化系统
```typescript
import { 
  initializeIntelligentServices,
  getFeedbackLoopService,
  getSuggestionEngine,
  getVoiceService 
} from './services';

// 初始化所有服务
await initializeIntelligentServices();

// 获取服务实例
const feedbackService = getFeedbackLoopService();
const suggestionEngine = getSuggestionEngine();
const voiceService = getVoiceService();
```

### 智能Agent调用
```typescript
import { callAgentIntelligent } from './services';

const response = await callAgentIntelligent(
  '我想设计一个可爱的猫咪IP形象',
  'director',
  {
    enableRAG: true,
    enableMemory: true,
    enableIntent: true
  }
);
```

### 语音交互
```typescript
// 开始语音识别
voiceService.startListening({
  onResult: (result) => {
    if (result.isFinal) {
      const command = voiceService.recognizeCommand(result.transcript);
      console.log('命令:', command.action);
    }
  }
});

// 语音播报
await voiceService.speak('已收到您的指令');
```

### 图像分析
```typescript
import { getImageUnderstandingService } from './services';

const imageService = getImageUnderstandingService();

// 分析参考图
const analysis = await imageService.analyzeImage(imageUrl);
console.log(`风格: ${analysis.style}, 氛围: ${analysis.mood}`);

// 生成设计建议
const suggestions = await imageService.generateDesignSuggestions(
  referenceUrl,
  'IP形象'
);
```

### 反馈收集
```typescript
// 提交显式反馈
feedbackService.submitExplicitFeedback(
  FeedbackType.EXPLICIT_LIKE,
  { outputId: 'design-123', agentType: 'designer' },
  { rating: 5, comment: '非常满意这个设计', tags: ['风格好', '色彩佳'] }
);

// 记录停留时间
feedbackService.recordDwellTime('design-123', 45000, context);

// 分析反馈
const analysis = feedbackService.analyzeFeedback();
console.log(`满意度: ${analysis.satisfactionRate * 100}%`);
```

### 主动建议
```typescript
const suggestions = await suggestionEngine.generateSuggestions({
  currentAgent: 'director',
  conversationStage: 'collecting',
  recentMessages: [{ role: 'user', content: '我想设计一个IP形象' }]
});

// 应用建议
suggestionEngine.applySuggestion(suggestions[0]);
```

## 🎯 应用场景

### 场景1：智能设计助手
```
用户："帮我设计一个可爱的猫咪IP"
系统：
  1. 意图识别 → CREATE_DESIGN
  2. RAG检索 → 找到相关案例
  3. 主动建议 → 推荐温馨风格
  4. 语音播报 → "好的，我来为您设计"
  5. 工作流执行 → 自动完成设计流程
```

### 场景2：参考图设计
```
用户：上传参考图
系统：
  1. 图像分析 → 识别风格、颜色、元素
  2. 生成建议 → "这是温馨治愈风格，建议采用类似风格"
  3. 创建任务 → 基于参考图生成设计
```

### 场景3：语音设计
```
用户："参考这张图的风格设计Logo"（上传图片）
系统：
  1. 语音识别 → 理解指令
  2. 图像分析 → 分析参考图
  3. 语音回复 → "已理解，我将参考温馨风格为您设计"
```

### 场景4：持续优化
```
用户：点踩不满意的设计
系统：
  1. 记录反馈 → 收集负面反馈
  2. 分析原因 → 识别问题模式
  3. 优化建议 → 生成改进方案
  4. 学习进化 → 避免类似问题
```

## 🔧 技术栈

### 核心AI能力
- **大语言模型**: 通义千问 (Qwen)
- **视觉模型**: 通义千问VL (Qwen-VL)
- **Embedding**: 本地实现 + API降级

### 浏览器API
- **Web Speech API**: 语音识别与合成
- **LocalStorage**: 数据持久化
- **Canvas**: 图像处理

### 前端框架
- **React**: UI框架
- **TypeScript**: 类型安全
- **Zustand**: 状态管理

## 📝 注意事项

1. **浏览器兼容性**:
   - 语音识别：Chrome、Edge、Safari支持较好
   - 建议使用HTTPS环境

2. **存储限制**:
   - LocalStorage约5-10MB限制
   - 向量数据自动清理旧数据

3. **API调用**:
   - 注意控制调用频率
   - 所有功能都有本地降级方案

4. **隐私保护**:
   - 用户数据本地存储
   - 反馈数据可导出清除
