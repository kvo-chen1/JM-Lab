# 津小脉Agent测试用例库

## 一、测试概述

### 1.1 测试目标
- 验证Agent各功能模块的正确性
- 检测交互连贯性问题
- 识别性能瓶颈和卡壳原因
- 确保异常处理的健壮性

### 1.2 测试范围
- 核心服务层（agentService, agentOrchestrator, llmService）
- 状态管理（useAgentStore）
- 任务队列（aiTaskQueueService, taskQueueManager）
- 智能服务（predictionService, suggestionEngine, workflowEngine）
- 错误处理（errorHandler）

## 六、问题汇总与优先级

### 6.1 已识别问题

| 问题ID | 问题描述 | 严重程度 | 影响范围 | 优先级 |
|--------|----------|----------|----------|--------|
| ISS-001 | 流式输出是模拟的，非真实流式 | 中 | 用户体验 | P1 |
| ISS-002 | 无任务超时机制 | 高 | 系统稳定性 | P0 |
| ISS-003 | 并发数固定，无法动态调整 | 中 | 性能 | P1 |
| ISS-004 | 行为记录无限增长 | 中 | 内存 | P1 |
| ISS-005 | 无网络状态监听 | 高 | 用户体验 | P0 |
| ISS-006 | 上下文可能过长导致Token超限 | 高 | 功能 | P0 |
| ISS-007 | 降级回复固定，缺乏上下文感知 | 中 | 用户体验 | P1 |
| ISS-008 | 轮询机制无最大重试次数 | 中 | 性能 | P1 |
| ISS-009 | localStorage容量限制 | 低 | 数据持久化 | P2 |
| ISS-010 | 消息过多时无自动清理 | 中 | 内存 | P1 |

### 6.2 优化建议

1. **立即处理（P0）**:
   - 添加任务超时机制
   - 实现网络状态监听
   - 添加上下文长度限制

2. **短期处理（P1）**:
   - 实现真实流式输出
   - 动态调整并发数
   - 限制行为记录数量
   - 优化降级回复

3. **长期处理（P2）**:
   - 使用IndexedDB替代localStorage
   - 实现消息自动清理
   - 添加更多恢复策略
