## ⏰ 时间格式化工具

### 核心 API

#### 1. 解析日期

```typescript
import { DateFormatter } from '@/utils/dateFormatter';

// 解析 ISO 字符串
const date1 = DateFormatter.parse('2024-01-15T10:30:00.000Z');

// 解析时间戳 (毫秒)
const date2 = DateFormatter.parse(1705315800000);

// 解析时间戳 (秒)
const date3 = DateFormatter.parse(1705315800);

// 解析 Date 对象
const date4 = DateFormatter.parse(new Date());

// 验证日期是否有效
const isValid = DateFormatter.isValid(date1);
```

#### 2. 格式化日期

```typescript
import { DateFormatter, DateFormat } from '@/utils/dateFormatter';

const date = new Date('2024-01-15T10:30:45.123Z');

// ISO 格式 (默认)
DateFormatter.format(date); 
// 输出："2024-01-15T10:30:45.123Z"

// 日期格式
DateFormatter.format(date, DateFormat.DATE);
// 输出："2024-01-15"

// 时间格式
DateFormatter.format(date, DateFormat.TIME);
// 输出："10:30:45"

// 日期时间格式
DateFormatter.format(date, DateFormat.DATETIME);
// 输出："2024-01-15 10:30:45"

// 中文格式
DateFormatter.format(date, DateFormat.CHINESE);
// 输出："2024 年 01 月 15 日 10:30"

// 时间戳 (毫秒)
DateFormatter.format(date, DateFormat.TIMESTAMP);
// 输出：1705315845123

// 时间戳 (秒)
DateFormatter.format(date, DateFormat.TIMESTAMP_SECONDS);
// 输出：1705315845
```

#### 3. 获取当前时间

```typescript
// ISO 格式 (推荐用于数据库存储)
const nowISO = DateFormatter.now();
// 输出："2024-01-15T10:30:00.000Z"

// 时间戳 (毫秒)
const nowMs = DateFormatter.nowTimestamp();
// 输出：1705315800000

// 时间戳 (秒)
const nowSec = DateFormatter.nowTimestampSeconds();
// 输出：1705315800
```

#### 4. 相对时间

```typescript
const postDate = new Date(Date.now() - 5 * 60 * 1000); // 5 分钟前

// 中文相对时间
DateFormatter.relativeTime(postDate, 'zh');
// 输出："5 分钟前"

// 英文相对时间
DateFormatter.relativeTime(postDate, 'en');
// 输出："5 minutes ago"

// 自动判断
DateFormatter.relativeTime(new Date());
// 输出："刚刚"

DateFormatter.relativeTime(new Date(Date.now() - 2 * 60 * 60 * 1000));
// 输出："2 小时前"

DateFormatter.relativeTime(new Date(Date.now() - 3 * 24 * 60 * 60 * 1000));
// 输出："3 天前"
```

#### 5. 日期比较和计算

```typescript
const date1 = new Date('2024-01-01');
const date2 = new Date('2024-01-15');

// 比较日期
DateFormatter.compare(date1, date2);
// 输出：-1 (date1 < date2)

DateFormatter.compare(date2, date1);
// 输出：1 (date2 > date1)

DateFormatter.compare(date1, date1);
// 输出：0 (相等)

// 计算差异
DateFormatter.diff(date1, date2, 'd');
// 输出：14 (天)

DateFormatter.diff(date1, date2, 'h');
// 输出：336 (小时)

// 日期加减
const newDate = DateFormatter.add(date1, 7, 'd');
// date1 + 7 天

const pastDate = DateFormatter.add(date1, -3, 'd');
// date1 - 3 天
```

#### 6. 特殊判断

```typescript
// 是否今天
DateFormatter.isToday(new Date()); // true
DateFormatter.isToday(new Date('2020-01-01')); // false

// 是否昨天
const yesterday = new Date();
yesterday.setDate(yesterday.getDate() - 1);
DateFormatter.isYesterday(yesterday); // true

// 是否未来
const tomorrow = new Date();
tomorrow.setDate(tomorrow.getDate() + 1);
DateFormatter.isFuture(tomorrow); // true

// 是否过去
DateFormatter.isPast(yesterday); // true
```

## 👥 用户同步服务

### 核心 API

#### 1. 初始化

```typescript
import { userSyncService } from '@/services/userSyncService';

// 在应用启动时初始化
userSyncService.init();
```

#### 2. 设置用户数据

```typescript
// 登录成功后设置用户数据
const userData = {
  id: 'user-123',
  username: 'zhangsan',
  email: 'zhangsan@example.com',
  avatar: 'https://example.com/avatar.jpg',
  bio: '这是一个开发者',
  location: '北京',
  website: 'https://zhangsan.dev',
};

// 立即同步到服务器
userSyncService.setUser(userData, {
  immediate: true,  // 立即同步
  notify: true,     // 通知其他组件
  persist: true,    // 保存到 localStorage
});

// 延迟同步 (适合频繁更新)
userSyncService.setUser(userData, {
  immediate: false, // 防抖同步 (300ms)
  notify: true,
  persist: true,
});
```

#### 3. 获取用户数据

```typescript
// 获取当前用户
const user = userSyncService.getUser();

// 获取用户 ID
const userId = userSyncService.getUserId();

// 检查是否已登录
if (userSyncService.getUserId()) {
  console.log('用户已登录');
}
```

#### 4. 更新用户数据

```typescript
// 更新头像
userSyncService.updateUser({
  avatar: 'https://example.com/new-avatar.jpg',
}, {
  immediate: false, // 延迟同步
  notify: true,
  persist: true,
});

// 更新资料
userSyncService.updateUser({
  bio: '新的个人简介',
  location: '上海',
  website: 'https://new-website.com',
});

// 更新设置
userSyncService.updateUser({
  settings: {
    theme: 'dark',
    language: 'zh',
    notifications: true,
  },
});
```

#### 5. 刷新用户数据

```typescript
// 从服务器刷新用户数据
const fetchUserFn = async () => {
  const response = await fetch('/api/user/me', {
    headers: {
      'Authorization': `Bearer ${token}`,
    },
  });
  
  if (response.ok) {
    const data = await response.json();
    return data.user;
  }
  
  return null;
};

const success = await userSyncService.refreshUser(fetchUserFn);
if (success) {
  console.log('用户数据已刷新');
}
```

#### 6. 清空用户数据 (登出)

```typescript
const handleLogout = () => {
  // 清空用户数据
  userSyncService.clear();
  
  // 清除 token
  localStorage.removeItem('token');
  
  // 跳转到登录页
  navigate('/login');
};
```

#### 7. 监听用户状态变化

```typescript
import { useEffect } from 'react';

// 在组件中监听用户更新
useEffect(() => {
  const unsubscribe = userSyncService.onUserUpdate((user) => {
    console.log('用户数据已更新', user);
    // 更新组件状态
    setUser(user);
  });
  
  // 清理监听器
  return unsubscribe;
}, []);

// 监听用户登出
useEffect(() => {
  const unsubscribe = userSyncService.onUserLogout(() => {
    console.log('用户已登出');
    // 清理用户相关数据
    setAuthenticated(false);
  });
  
  return unsubscribe;
}, []);
```

#### 8. 强制同步

```typescript
// 立即同步到服务器 (不等待防抖)
await userSyncService.forceSync();
```

## ❓ 常见问题

### Q1: 为什么要使用 ISO 格式而不是时间戳？

**A:** ISO 格式有以下优势:
- ✅ 可读性更好，便于调试
- ✅ 时区信息明确 (带 Z 表示 UTC)
- ✅ 数据库支持好 (TIMESTAMPTZ)
- ✅ 跨语言/跨平台兼容性好
- ✅ 便于日志分析和排查问题

### Q2: 多标签页同步有延迟吗？

**A:** 
- storage 事件是实时的，通常在 100ms 内完成同步
- 同步过程: Tab A 更新 → localStorage → storage 事件 → Tab B 接收
- 如果 Tab B 未响应，可能是:
  - Tab B 未初始化 userSyncService
  - 浏览器限制了 storage 事件
  - 跨域问题

### Q3: 防抖会影响数据一致性吗？

**A:** 
- 不会。防抖只影响服务器同步
- localStorage 是立即更新的
- 多标签页同步也是实时的
- 服务器同步延迟不影响用户体验

### Q4: 如何处理离线情况？

**A:** 
- userSyncService 会自动缓存更新到 localStorage
- 网络恢复后，会自动同步到服务器
- 可以调用 `forceSync()` 手动触发同步

### Q5: 时间格式化性能如何？

**A:** 
- DateFormatter 是纯函数，无状态
- 解析和格式化都在毫秒级完成
- 不会造成性能瓶颈
- 可以安全地在渲染函数中使用

### Q6: 用户数据会过期吗？

**A:** 
- localStorage 中的数据不会自动过期
- 建议定期刷新用户数据:
  ```typescript
  // 每小时刷新一次
  setInterval(() => {
    userSyncService.refreshUser(fetchUserFn);
  }, 60 * 60 * 1000);
  ```

### Q7: 如何调试同步问题？

**A:** 
- 开启调试日志:
  ```typescript
  // 控制台会输出详细日志
  console.log('[UserSyncService] Debug info:', {
    user: userSyncService.getUser(),
    storage: localStorage.getItem('user_data'),
  });
  ```
- 检查 storage 事件:
  ```typescript
  window.addEventListener('storage', (e) => {
    console.log('Storage event:', e.key, e.newValue);
  });
  ```
