# 数据一致性问题修复报告

## 修复概述

本次修复针对项目中存在的两个重大数据一致性问题：
1. **时间戳格式不统一**
2. **用户状态同步延迟**

通过创建全局时间格式化工具类和用户数据同步服务，从根本上解决了这些问题。

---

## 问题一：时间戳格式不统一

### 问题分析

#### 具体表现
1. 项目中存在多种时间格式混用的情况：
   - `Date.now()` - 毫秒级时间戳数字
   - `new Date().toISOString()` - ISO 8601 字符串
   - `new Date()` - Date 对象
   - 自定义格式字符串

2. 数据库字段类型不统一：
   - 部分表使用 `timestamp` 类型
   - 部分表使用 `text` 或 `varchar` 存储时间

3. 前后端交互时格式转换不一致

#### 触发条件
- 创建新记录时，不同开发人员使用不同的时间格式
- 查询数据时，时间比较可能因为格式不同而失败
- 跨系统数据同步时，时间解析错误

#### 根本原因
- 缺少统一的时间格式化工具
- 开发规范未明确时间格式要求
- 数据库 schema 定义不够严格

### 修复方案

#### 1. 创建全局时间格式化工具类

**文件位置**: `src/utils/dateFormatter.ts`

**核心功能**:
- 统一使用 ISO 8601 格式作为标准时间格式
- 支持多种时间格式的解析和转换
- 提供丰富的日期操作工具函数
- 支持国际化相对时间格式化

**主要 API**:

```typescript
// 解析任意格式的日期
DateFormatter.parse(dateValue: any): Date | null

// 格式化为指定格式
DateFormatter.format(dateValue: any, format: DateFormat): string | number | null

// 获取当前时间 (ISO 格式)
DateFormatter.now(): string

// 获取当前时间戳 (毫秒)
DateFormatter.nowTimestamp(): number

// 获取相对时间描述
DateFormatter.relativeTime(dateValue: any, locale: 'zh' | 'en'): string

// 日期比较
DateFormatter.compare(date1: any, date2: any): number

// 计算日期差异
DateFormatter.diff(date1: any, date2: any, unit: string): number

// 日期加减
DateFormatter.add(dateValue: any, amount: number, unit: string): Date | null

// 日期验证
DateFormatter.isValid(dateValue: any): boolean

// 判断是否今天/昨天/未来/过去
DateFormatter.isToday(dateValue: any): boolean
DateFormatter.isYesterday(dateValue: any): boolean
DateFormatter.isFuture(dateValue: any): boolean
DateFormatter.isPast(dateValue: any): boolean
```

#### 2. 使用规范

**数据库存储**:
```typescript
// ✅ 推荐：使用 ISO 格式
const record = {
  created_at: DateFormatter.now(), // "2024-01-15T10:30:00.000Z"
  updated_at: DateFormatter.now(),
};

// ❌ 不推荐：使用时间戳数字
const record = {
  created_at: Date.now(), // 1705315800000
};
```

**前端展示**:
```typescript
// 显示相对时间
const relativeTime = DateFormatter.relativeTime(post.created_at, 'zh');
// 输出："3 小时前"

// 显示格式化日期
const displayDate = DateFormatter.format(post.created_at, DateFormat.DATETIME);
// 输出："2024-01-15 10:30:00"

// 显示中文格式
const chineseDate = DateFormatter.format(post.created_at, DateFormat.CHINESE);
// 输出："2024 年 01 月 15 日 10:30"
```

**时间计算**:
```typescript
// 计算过期时间
const expireDate = DateFormatter.add(new Date(), 7, 'd');

// 比较时间
if (DateFormatter.compare(startDate, endDate) > 0) {
  console.error('开始时间不能晚于结束时间');
}

// 计算天数差异
const daysDiff = DateFormatter.diff(startDate, endDate, 'd');
```

### 迁移指南

## 问题二：用户状态同步延迟

### 问题分析

#### 具体表现
1. 登录后用户数据更新不及时
2. localStorage 与数据库数据不一致
3. 多标签页用户状态不同步
4. 用户信息更新后其他组件未刷新

#### 触发条件
- 用户登录后立即访问需要用户数据的页面
- 修改用户信息 (如头像) 后其他页面未更新
- 多标签页同时操作时状态不一致

#### 根本原因
- 缺少统一的用户数据管理中心
- localStorage 更新后未触发事件通知
- 组件间数据同步机制不健全
- 没有防抖和节流机制，导致频繁同步

### 修复方案

#### 1. 创建用户数据同步服务

**文件位置**: `src/services/userSyncService.ts`

**核心功能**:
- 统一的用户数据管理
- 自动同步到本地存储
- 多标签页同步支持
- 防抖同步机制
- 定期同步检查
- 事件通知机制

**主要 API**:

```typescript
// 初始化服务
userSyncService.init();

// 设置用户数据
userSyncService.setUser(userData: UserData, options: SyncOptions);

// 获取用户数据
userSyncService.getUser(): UserData | null;

// 获取用户 ID
userSyncService.getUserId(): string | null;

// 更新用户部分字段
userSyncService.updateUser(partial: Partial<UserData>, options: SyncOptions);

// 刷新用户数据
await userSyncService.refreshUser(fetchUserFn);

// 清空用户数据 (登出)
userSyncService.clear();

// 强制同步到服务器
await userSyncService.forceSync();

// 监听用户更新
const unsubscribe = userSyncService.onUserUpdate((user) => {
  console.log('用户数据已更新', user);
});

// 监听用户登出
const unsubscribe = userSyncService.onUserLogout(() => {
  console.log('用户已登出');
});
```

#### 2. 同步策略

**立即同步**:
```typescript
// 登录时立即同步
userSyncService.setUser(userData, {
  immediate: true,  // 立即同步到服务器
  notify: true,     // 通知其他组件
  persist: true,    // 保存到 localStorage
});
```

**延迟同步 (防抖)**:
```typescript
// 用户信息更新时延迟同步
userSyncService.updateUser({ avatar: newAvatar }, {
  immediate: false, // 延迟同步 (防抖 300ms)
  notify: true,
  persist: true,
});
```

**多标签页同步**:
```typescript
// 自动监听 storage 事件
// 当其他标签页更新用户数据时，自动同步到当前标签页
window.addEventListener('storage', (event) => {
  if (event.key === 'user_data') {
    // 自动处理同步
  }
});
```

### 使用指南

#### 1. 在应用启动时初始化

```typescript
// src/main.tsx 或 src/App.tsx
import { userSyncService } from '@/services/userSyncService';

// 应用启动时初始化
userSyncService.init();
```

#### 2. 登录时设置用户数据

```typescript
// 登录成功后
const handleLogin = async (credentials) => {
  const response = await login(credentials);
  const userData = response.user;
  
  // 设置用户数据并立即同步
  userSyncService.setUser(userData, {
    immediate: true,
    notify: true,
    persist: true,
  });
  
  navigate('/home');
};
```

#### 3. 更新用户信息

```typescript
// 更新头像
const handleAvatarUpdate = async (newAvatar) => {
  // 乐观更新
  userSyncService.updateUser({ avatar: newAvatar }, {
    immediate: false, // 延迟同步
    notify: true,
    persist: true,
  });
  
  // 后台同步到服务器
  await uploadAvatar(newAvatar);
};
```

#### 4. 监听用户状态变化

```typescript
// 在组件中监听用户状态
useEffect(() => {
  const unsubscribe = userSyncService.onUserUpdate((user) => {
    // 用户数据已更新，刷新组件
    setUser(user);
  });
  
  return unsubscribe;
}, []);
```

#### 5. 登出时清空数据

```typescript
const handleLogout = () => {
  // 清空用户数据
  userSyncService.clear();
  
  // 清除 token 等
  localStorage.removeItem('token');
  
  navigate('/login');
};
```

### B. 常见问题

**Q: 为什么要使用 ISO 格式而不是时间戳？**
A: ISO 格式可读性更好，时区信息明确，数据库支持好，便于调试。

**Q: 多标签页同步有延迟吗？**
A: storage 事件是实时的，通常在 100ms 内完成同步。

**Q: 防抖会影响数据一致性吗？**
A: 不会。本地存储是立即更新的，服务器同步延迟不影响用户体验。

**Q: 如何处理离线情况？**
A: 服务会自动缓存更新，网络恢复后自动同步。
