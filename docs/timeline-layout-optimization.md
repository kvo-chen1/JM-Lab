# 时间线布局优化设计方案

## 当前问题分析

当前时间线采用单条水平布局，节点按时间顺序从左到右排列。当节点数量增多时（如9个节点），会出现以下问题：
1. 时间线过长，需要大量水平滚动
2. 节点超出可视区域，浏览困难
3. 空间利用率低，大量空白区域
4. 移动端体验差，难以操作

---

## 方案一：双行交错时间线（推荐方案）

### 设计思路
将时间线改为上下两行交错排列，奇数节点在上行，偶数节点在下行，形成"Z"字形或波浪形时间线。

```
时间流向 →
[节点1]    [节点3]    [节点5]    [节点7]    [节点9]
    ↘          ↘          ↘          ↘
      [节点2]    [节点4]    [节点6]    [节点8]
```

### 优点
1. ✅ **空间利用率高**：同样宽度可展示约2倍节点
2. ✅ **视觉层次清晰**：上下交错形成自然的阅读流
3. ✅ **保持时间顺序**：从左到右、从上到下的阅读顺序
4. ✅ **易于实现**：只需修改Y坐标计算逻辑
5. ✅ **响应式友好**：移动端可改为垂直布局

### 缺点
1. ❌ 需要垂直滚动（但比水平滚动更自然）
2. ❌ 节点连接线需要特殊处理

### 实现要点
- 节点宽度：220px，节点间距：40px
- 上行Y坐标：0，下行Y坐标：180px
- 连接线使用贝塞尔曲线连接上下行
- 添加时间轴主线贯穿整个布局

## 方案二：瀑布流卡片布局

### 设计思路
采用Pinterest风格的瀑布流布局，节点按时间顺序以卡片形式排列，自动填充空间。

```
┌─────────┐ ┌─────────┐ ┌─────────┐
│ 节点1   │ │ 节点2   │ │ 节点3   │
│         │ │         │ │         │
└─────────┘ └─────────┘ └─────────┘
┌─────────┐ ┌─────────┐ ┌─────────┐
│ 节点4   │ │ 节点5   │ │ 节点6   │
│         │ │         │ │         │
└─────────┘ └─────────┘ └─────────┘
```

### 优点
1. ✅ **空间利用率最高**：无固定行高，自适应内容
2. ✅ **视觉丰富**：卡片大小不一，更有设计感
3. ✅ **响应式极佳**：自动适应不同屏幕宽度
4. ✅ **支持懒加载**：适合大量节点场景

### 缺点
1. ❌ **时间顺序不直观**：需要额外的日期标识
2. ❌ **连接线难以实现**：节点位置不固定
3. ❌ **实现复杂**：需要计算每个卡片位置

### 实现要点
- 使用CSS Grid或Masonry布局
- 每个卡片显示时间戳
- 按日期分组显示
- 添加"加载更多"功能

## 方案三：时间轴+分组折叠布局

### 设计思路
按时间段（如按天、按周）将节点分组，每组可折叠展开，默认只显示每组的关键节点。

```
2026年2月15日 ▼
├── [节点1] [节点2] [节点3] ...
2026年2月14日 ▶
├── (3个节点)
2026年2月13日 ▶
├── (2个节点)
```

### 优点
1. ✅ **信息密度高**：大量节点也能快速概览
2. ✅ **易于导航**：可按日期快速定位
3. ✅ **层次分明**：时间分组清晰
4. ✅ **用户可控**：可展开/折叠查看详情

### 缺点
1. ❌ 需要额外点击操作查看全部
2. ❌ 不适合展示连续创作过程
3. ❌ 实现复杂度中等

### 实现要点
- 按日期自动分组
- 每组显示节点缩略图
- 支持展开/折叠动画
- 添加日期导航栏

## 最终推荐：方案一（双行交错时间线）

### 推荐理由
1. **平衡性最佳**：在保持时间顺序清晰的同时，大幅提升空间利用率
2. **符合用户习惯**：水平时间线+垂直滚动的组合最自然
3. **实现成本可控**：基于现有代码修改，不需要重构
4. **扩展性好**：未来可轻松扩展为三行或更多行

### 具体实现规划

#### 1. 布局算法修改
```typescript
// 双行交错布局
private calculateDoubleRowLayout(nodes: MindNode[]): NodePosition[] {
  const positions: NodePosition[] = [];
  const itemWidth = 260;
  const rowHeight = 200;
  
  nodes.forEach((node, index) => {
    const row = index % 2; // 0 = 上行, 1 = 下行
    const col = Math.floor(index / 2);
    
    positions.push({
      nodeId: node.id,
      x: col * itemWidth,
      y: row * rowHeight,
      level: row,
    });
  });
  
  return positions;
}
```

#### 2. 视觉设计
- **上行节点**：标准卡片样式
- **下行节点**：略微缩小的卡片，或不同背景色
- **连接线**：使用曲线连接上下行节点
- **时间轴**：中间添加垂直时间轴线

#### 3. 交互优化
- 鼠标悬停显示完整信息
- 点击节点高亮相关连接线
- 支持拖拽调整节点位置

#### 4. 响应式适配
- **桌面端**：双行布局
- **平板端**：双行布局，节点宽度减小
- **移动端**：改为垂直单列布局
