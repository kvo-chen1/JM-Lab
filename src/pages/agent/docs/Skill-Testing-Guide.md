# Skill 架构测试指南

## 测试页面访问

访问 `/agent/skill-test` 进入 Skill 架构测试中心。

## 测试功能介绍

### 1. Agent 选择器

页面顶部可以切换不同的 Agent，观察 Skill 列表的变化：

- **设计总监 (director)** - 拥有编排、分析类 Skill
- **品牌设计师 (designer)** - 拥有图像生成、设计类 Skill
- **插画师 (illustrator)** - 拥有图像生成、插画类 Skill
- **文案策划 (copywriter)** - 拥有文本生成类 Skill
- **动画师 (animator)** - 拥有视频生成类 Skill
- **研究员 (researcher)** - 拥有分析、研究类 Skill

**测试步骤**：
1. 点击顶部 "当前 Agent" 下拉菜单
2. 选择不同的 Agent
3. 观察左侧 Skill 列表的变化

### 2. 自动模式测试

自动模式会根据用户输入自动识别意图并匹配最佳 Skill。

**测试用例**：

```
测试 1: 图像生成
输入: "帮我画一个可爱的猫咪"
预期: 匹配到 image-generation Skill

测试 2: 文案生成
输入: "写一段产品描述"
预期: 匹配到 text-generation Skill

测试 3: 需求分析
输入: "分析一下这个设计需求"
预期: 匹配到 requirement-analysis Skill
```

**测试步骤**：
1. 选择 Agent（如 designer）
2. 在测试面板中选择 "自动模式"
3. 输入测试消息
4. 点击 "执行测试"
5. 查看匹配结果和执行日志

### 3. 手动模式测试

手动模式允许直接执行指定的 Skill。

**测试步骤**：
1. 在左侧 Skill 列表中选择一个 Skill
2. 在测试面板中选择 "手动模式"
3. 输入测试消息
4. 点击 "执行测试"

### 4. Skill 调试工具

点击右下角的 🐛 图标打开调试工具。

**功能标签页**：

#### 注册表
- 查看所有注册的 Skill
- 查看当前 Agent 可用的 Skill（绿色标记）
- 查看 Skill 统计信息

#### 匹配测试
- 输入测试消息
- 查看意图识别结果
- 查看匹配的 Skill 列表及分数
- 直接执行匹配的 Skill

#### 执行日志
- 查看执行状态
- 查看详细的执行结果
- 查看执行日志历史

## 测试场景

### 场景 1: 验证 Agent-Skill 映射

**目的**: 验证不同 Agent 拥有不同的 Skill

**步骤**:
1. 选择 "设计总监" Agent
2. 观察左侧 Skill 列表，应该看到：意图识别、需求分析、任务编排等
3. 切换到 "品牌设计师" Agent
4. 观察左侧 Skill 列表，应该看到：意图识别、图像生成、风格推荐等
5. 比较两个列表，确认 Skill 不同

### 场景 2: 验证 Skill 匹配过滤

**目的**: 验证 SkillMatcher 会根据 Agent 过滤 Skill

**步骤**:
1. 选择 "文案策划" Agent
2. 打开调试工具，切换到 "匹配测试" 标签
3. 输入: "画一只狗"
4. 点击测试
5. 观察匹配结果：应该匹配到 text-generation，而不是 image-generation

### 场景 3: 验证意图识别

**目的**: 验证意图识别 Skill 能正确识别用户意图

**步骤**:
1. 选择任意 Agent
2. 打开调试工具，切换到 "匹配测试" 标签
3. 输入不同的测试消息：
   - "设计一个 Logo" → 应该识别为 image-generation
   - "写一段文案" → 应该识别为 text-generation
   - "分析一下市场" → 应该识别为 requirement-analysis

### 场景 4: 验证 Skill 执行

**目的**: 验证 Skill 能正确执行

**步骤**:
1. 选择 "品牌设计师" Agent
2. 在左侧选择 "图像生成" Skill
3. 选择手动模式
4. 输入: "一只在草地上玩耍的金毛犬"
5. 点击执行
6. 观察执行结果和生成的图片

## 常见问题排查

### 问题 1: Skill 列表为空

**可能原因**:
- Skill 未正确注册
- 当前 Agent 没有可用的 Skill

**排查步骤**:
1. 打开调试工具，查看 "注册表" 标签
2. 检查是否有 Skill 显示
3. 如果没有，检查 useSkill 的初始化配置

### 问题 2: 意图识别失败

**可能原因**:
- 意图识别 Skill 未注册
- 输入消息格式问题

**排查步骤**:
1. 确认 IntentRecognitionSkill 在 Skill 列表中
2. 检查输入消息是否为空
3. 查看执行日志中的错误信息

### 问题 3: Skill 匹配不准确

**可能原因**:
- 意图识别结果不准确
- Skill 的 canHandle 方法实现问题

**排查步骤**:
1. 使用调试工具的 "匹配测试" 查看意图识别结果
2. 检查匹配分数和置信度
3. 调整 Skill 的匹配逻辑

### 问题 4: Agent 切换后 Skill 未更新

**可能原因**:
- Skill 列表未刷新

**排查步骤**:
1. 点击 "刷新 Skill 列表" 按钮
2. 检查 useSkill 是否正确传入 currentAgent

## 性能测试

### 测试 Skill 执行时间

**步骤**:
1. 多次执行同一个 Skill
2. 观察右侧结果面板的执行时间
3. 计算平均执行时间

### 测试并发执行

**步骤**:
1. 快速连续发送多个测试消息
2. 观察系统是否能正确处理
3. 查看日志中的处理顺序

## 调试技巧

### 查看详细的 Skill 信息

在调试工具的 "注册表" 标签中：
- 点击任意 Skill 查看详细信息
- 查看 Skill 的 supportedAgents
- 查看 Skill 的元数据

### 追踪执行流程

在调试工具的 "执行日志" 标签中：
- 查看每个步骤的执行时间
- 查看意图识别结果
- 查看 Skill 匹配结果
- 查看最终执行结果

### 测试边界情况

**测试空输入**:
- 输入空字符串
- 观察系统如何处理

**测试超长输入**:
- 输入超过 1000 字的文本
- 观察系统如何处理

**测试特殊字符**:
- 输入包含 emoji、特殊符号的文本
- 观察系统如何处理

## 下一步
